# Access Matrix

## Practical Access Areas

The current access matrix should be reasoned about in these buckets:

- shared workspace access
- solution-module access
- shared-data maintenance
- admin controls
- approvals

## Current High-Level Expectations

- most active members can view platform summaries and shared data
- admin-like roles control billing, settings, and sensitive platform changes
- approval-capable roles should be able to review approval-gated requests
- solution modules may be enabled or disabled per organization regardless of role

## Rule

Do not treat the old workflow pages as the primary access taxonomy anymore. They are retained for compatibility, but the current access model should be explained in terms of the live grouped platform structure.
