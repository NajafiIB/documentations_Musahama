# Implementation Guides

This folder stores bounded implementation instructions derived from approved change requests.

## Purpose
Use an implementation guide to define:
- what the coding agent must read first
- which files are allowed to change
- which files must not change
- what architecture rules must be respected
- what the implementation sequence is
- how success will be verified

## Naming
Use `IMP-YYYY-NNN.md`.

## Rules
1. do not create an implementation guide without a clear change request
2. every guide must define allowed and forbidden files
3. every guide must define acceptance criteria
4. every guide must define what the coding agent should return
