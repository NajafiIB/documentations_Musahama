# Reviews

This folder stores structured review documents for implementation output.

## Purpose
Use a review to verify:
- correct layer ownership
- correct source-of-truth usage
- correct module and feature boundaries
- workflow integrity
- docs alignment
- risk handling

## Naming
Use `RV-YYYY-NNN.md`.

## Rules
1. every non-trivial implementation should have a review
2. a review must call out architecture violations clearly
3. a review must state pass, rework-required, or failed
4. a review should reference actual changed files and behavior
