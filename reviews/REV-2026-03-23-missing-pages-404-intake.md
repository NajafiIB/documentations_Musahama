# Review — Missing Pages / 404 Intake (2026-03-23)

## Scope
Define post-stage tasks to close remaining 404s for designed-but-not-wired workspace pages and dashboard action links.

## Problem
Users can click dashboard/module actions (including mandate-related entry points) and encounter 404 pages because routes exist in product design but are not fully implemented or wired.

## Target Outcome
- Every canonical module route resolves to at least a thin, non-404 page.
- Dashboard action buttons (especially mandate flow entry) navigate to valid routes.
- Route-availability metadata matches actual route implementation state.
