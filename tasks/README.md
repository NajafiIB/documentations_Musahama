# Tasks

This folder stores atomic execution units created from an implementation guide.

## Purpose
Use a task file to define:
- one bounded objective
- the owning layer
- allowed and forbidden files
- prerequisites
- acceptance criteria
- the expected implementation output

## Naming
Use `TASK-YYYY-NNN-01.md`.

## Rules
1. tasks should be atomic
2. one task should ideally map to one PR
3. tasks must link back to the parent change request and implementation guide
4. task ordering and active state should be mirrored in the parent case manifest
