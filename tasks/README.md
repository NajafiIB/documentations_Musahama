# Tasks

This folder stores atomic execution units created from an implementation guide.

## Purpose
Use tasks to define a bounded objective, the main owning layer, prerequisites, acceptance criteria, and the expected output from the implementer.

## Naming
Use `TASK-YYYY-NNN-01.md`.

## Rules
1. tasks should be atomic
2. one task should ideally map to one PR
3. tasks must link back to the parent change request and implementation guide
